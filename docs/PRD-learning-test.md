# PRD: 학습 유형 테스트 페이지 개선

**담당 페이지:** `/learning-test`  
**파일 위치:** `src/pages/LearningTypeTest.tsx`

---

## 1. 개요

학습 유형 테스트는 5가지 질문을 통해 자녀의 학습 성향을 진단하고, 결과를 프로필에 저장할 수 있는 기능이다.  
디자인은 온보딩 페이지(`OnboardingLayout`, `StepCard`)의 시각 언어를 그대로 따른다.

---

## 2. 학습 유형 정의 (변경 없음)

기존 5가지 유형과 채점 로직을 그대로 유지한다.

| Key | 한국어 이름 | 이모지 | 색상 |
|---|---|---|---|
| `competitive` | 경쟁 성취형 | 🏆 | Yellow |
| `explorer` | 탐구 사고형 | 🔍 | Blue |
| `systematic` | 체계 반복형 | 📋 | Green |
| `social` | 사교 협동형 | 👥 | Purple |
| `independent` | 자기주도형 | 🎯 | Rose |

**채점 방식:** 5개 질문 각각의 선택지에 매핑된 유형 키를 누적 집계, 가장 많이 선택된 유형이 결과

---

## 3. 화면 구성

### 3.1 인트로 화면 (Step 0)

#### 디자인
- 온보딩 페이지와 동일한 레이아웃/화면 크기 사용 (`OnboardingLayout` 또는 동일 스타일 적용)
- 배경 그라디언트, 카드 스타일, 타이포그래피 모두 온보딩 기준

#### 콘텐츠

```
[소제목] 학습 유형 테스트

[h3] 우리 아이 학습 성향 분석

[소문구] 5가지 질문으로 아이의 학습 성향을 파악하고,
         성향에 맞는 학원 선택 가이드를 받아보세요.

[안내 문구 - h3 아래 작은 글씨]
"검사 결과는 프로필에 저장하여 학원 추천에 활용할 수 있어요."
```

- 5가지 유형 미리보기 카드 (이모지 + 유형명 + 한 줄 설명)
- [테스트 시작하기 →] 버튼 + "약 1분 소요 · 총 5문항" 보조 문구

---

### 3.2 질문 화면 (Step 1–5)

#### 디자인
- 온보딩 `StepCard` 컴포넌트와 동일한 선택지 카드 스타일
- 상단 진행 바 (온보딩과 동일 스타일, 5구간)
- 선택 시 강조 border + 배경색 전환 (기존 로직 유지)

#### 콘텐츠
- 기존 `QUESTIONS` 배열 그대로 사용 (변경 없음)
- 선택지는 5개 카드, 각각 유형 키에 매핑

---

### 3.3 결과 화면 (분석 완료)

#### 레이아웃
온보딩과 동일한 화면 크기 내에서 2-column 구성:

```
┌─────────────────────────────┬──────────────────────┐
│  [좌] 결과 카드               │  [우] 유형 목록 패널   │
│  - 유형 이름 + 이모지          │  - 5가지 유형 리스트   │
│  - 설명                      │  - 클릭 시 유형 변경   │
│  - 강점 배지                  │                      │
│  - 학원 선택 가이드 (3가지)     │  [우] 프로필 저장 패널  │
│  - 주의사항                   │  - 프로필에 추가 버튼   │
│                             │  - 추가하지 않기 버튼   │
└─────────────────────────────┴──────────────────────┘
```

#### 좌측: 결과 카드
- 현재 선택된 유형(기본값: 테스트 결과)을 표시
- 유형 색상 그라디언트 배경
- 유형명, 이모지, 설명, 강점 배지(3개), 학원 찾기 가이드(3줄), 주의사항

#### 우측 상단: 5가지 유형 보기 패널
- 5가지 유형을 리스트로 표시
- 클릭 시 좌측 결과 카드의 내용을 해당 유형으로 **즉시 교체** (state 변경)
- 현재 선택된 유형에 강조 표시 (배경색 + "선택됨" 뱃지)
- 테스트 결과 유형에는 "우리 아이" 뱃지 유지

#### 우측 하단: 프로필 저장 패널

```
[패널 제목] 프로필에 저장하기

[설명] 선택한 유형을 프로필에 추가하면 학원 추천 시 반영돼요.

[버튼 1 - Primary]  "○○형을 프로필에 추가하기 →"
[버튼 2 - Secondary] "프로필에 추가하지 않기"
```

- 버튼 1 클릭 → 현재 선택된 유형을 프로필/스토어에 저장 → 성공 토스트 or 다음 화면 이동
- 버튼 2 클릭 → 저장 없이 다음 단계로 이동 (`/academies` 또는 이전 경로)
- "현재 선택된 유형"은 우측 유형 목록에서 선택한 유형 기준 (테스트 결과가 아닐 수 있음)

## 4. 상태 관리

### 추가될 상태 (LearningTypeTest 내부)

| 상태 | 타입 | 설명 |
|---|---|---|
| `step` | `'intro' \| 'quiz' \| 'result'` | 현재 화면 단계 (기존) |
| `answers` | `TypeKey[]` | 질문별 선택 답안 (기존) |
| `resultType` | `TypeKey` | 테스트 채점 결과 유형 (기존) |
| `selectedType` | `TypeKey` | 결과 화면에서 현재 선택된 유형 (신규) |

- `selectedType` 초기값은 `resultType`
- 유형 목록 클릭 시 `selectedType` 업데이트
- 프로필 추가 버튼은 `selectedType` 기준으로 동작

### 프로필 저장

- `onboardingStore` 또는 별도 `profileStore`에 `learningType: TypeKey` 필드 추가
- 저장 시 해당 필드에 `selectedType` 값 기입
- 마이페이지 등에서 이 값을 읽어 "학습 유형: 체계 반복형" 형태로 노출 가능

## 5. 디자인 기준 (온보딩 참조)

| 항목 | 온보딩 기준 적용 |
|---|---|
| 컨테이너 너비 | `max-w-lg` (온보딩과 동일) |
| 배경 | **비디오 배경 + 그라디언트 오버레이** (아래 참조) |
| 카드 | `rounded-2xl`, `border-2`, `shadow` |
| 선택 상태 | `border-primary` + 배경 강조 |
| 버튼 Primary | 초록색 배경 `bg-primary` (`hsl(130 79% 41%)`) |
| 버튼 Secondary | 흰 배경 + 초록색 border/텍스트 |
| 진행 바 | 온보딩 5구간 바와 동일 스타일 |
| 폰트 | 온보딩 동일 (소제목, h3, body) |

### 배경 구현 상세

온보딩 `OnboardingLayout`과 동일한 방식을 적용한다.

```
┌──────────────────────────────────────┐
│  <video> /correctvideo.mp4           │  ← absolute, inset-0, object-cover
│  <div> bg-gradient-to-b             │  ← absolute, from-slate-50/90 to-white/90
│  <div> 콘텐츠                        │  ← relative z-10
└──────────────────────────────────────┘
```

- `<video autoPlay loop muted playsInline>` → `src="/correctvideo.mp4"`
- 오버레이: `bg-gradient-to-b from-slate-50/90 to-white/90`
- 결과적으로 배경 영상이 흐릿하게 비치며 콘텐츠 가독성 확보
- 인트로/질문/결과 모든 step에서 동일 배경 유지 (배경은 고정, 콘텐츠만 전환)

```
[인트로 화면]
  → 테스트 시작하기 클릭
    → [질문 1/5] 선택
      → [질문 2/5] ...
        → [질문 5/5]
          → 채점 → [결과 화면]
            → 유형 목록에서 유형 선택 가능 (좌측 카드 실시간 업데이트)

              → "추가하지 않기" 클릭 → /academies 이동
```

## 8. 확정 사항

### 8-1. 저장 후 이동 경로
- 프로필에 추가하기/추가하지 않기 → 모두 `/academies`로 이동

### 8-2. 이미 저장된 유형이 있을 때 처리
결과 화면 진입 시 스토어에 `learningType`이 이미 저장되어 있으면:
- 우측 유형 목록에서 기존 저장 유형 옆에 **"저장됨"** 뱃지 노출
- 프로필 저장 패널 버튼 텍스트를 **"○○형으로 변경하기 →"** 로 교체 (새로 선택한 유형이 기존과 다를 경우)
- 기존과 동일한 유형이 선택된 경우 버튼 텍스트: **"현재 저장된 유형이에요"** (비활성)

### 8-3. 마이페이지 — 학습 유형 노출 및 변경

**위치:** `MyPage.tsx` 좌측 "내 카드" (프로필 그라디언트 카드) 하단에 추가

**현재 카드 구조 (`lg:col-span-1`):**
```
┌──────────────────────────────────────┐
│  [아이콘] 홍수인                       │  ← 기존 유지
│           0208suin@naver.com         │  ← 기존 유지
├──────────────────────────────────────┤
│  학습 유형                            │  ← 신규 추가 (하단 구분선 + 섹션)
│  [select] 체계 반복형 ▾               │
└──────────────────────────────────────┘
```

**구현 상세:**
- 카드 내부 하단에 구분선(`border-t border-white/20`) + "학습 유형" 레이블 + `<select>` 추가
- `<select>` 옵션: 5가지 유형 전체 + 빈 값("미설정")
- 변경 즉시 스토어 + (선택적으로) Supabase에 저장
- 유형 미설정 시 select에 "아직 테스트하지 않았어요" 플레이스홀더 표시
- 학습유형 테스트 페이지 링크 버튼 ("학습 유형 테스트하기 →") 을 select 아래에 작게 노출
